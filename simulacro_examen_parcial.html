Solución · IMC con clasificación (3 pts)

🎯 ¿Qué nos pidieron?

Calcular el IMC y clasificar a la persona en 4 categorías. Es un caso clásico de SI anidado en cascada con 4 niveles sobre un valor calculado (no leído directamente).

🧠 Razonamiento

Primero calculo IMC (proceso secuencial), luego aplico la cascada de mayor a menor (Obesidad primero, después Sobrepeso, después Normal, último Bajo peso por defecto):

  • IMC ≥ 30 → Obesidad
  • IMC ≥ 25 (pero < 30, gracias al SINO) → Sobrepeso
  • IMC ≥ 18.5 (pero < 25) → Normal
  • Caso por defecto → Bajo peso

🏷️ Leyenda

  • peso = peso de la persona en kg (entrada, Real)
  • estatura = estatura en metros (entrada, Real)
  • IMC = índice calculado (intermedia, Real)
  • clasif = mensaje de clasificación (salida, Cadena)

🪜 Pseudocódigo

INICIO
    Real: peso, estatura, IMC
    Cadena: clasif
    Leer peso, estatura
    IMC = peso / (estatura ^ 2)
    SI IMC >= 30
        clasif = "Obesidad"
    SINO
        SI IMC >= 25
            clasif = "Sobrepeso"
        SINO
            SI IMC >= 18.5
                clasif = "Normal"
            SINO
                clasif = "Bajo peso"
            FIN_SI
        FIN_SI
    FIN_SI
    Mostrar "IMC: ", IMC
    Mostrar "Clasificación: ", clasif
FIN

Solución · Tarifa eléctrica (4 pts)

🎯 ¿Qué nos pidieron?

Igual que el problema de descuentos por kilos de la PC2, pero ahora cada nivel tiene DOS valores (tarifa y cargo fijo). Es la misma estructura de SI anidado en cascada, solo que dentro de cada rama se asignan dos variables.

🧠 Razonamiento

Aplico orden de mayor a menor:

  1. Consumo ≥ 600: tarifa 0.80, cargo 18
  2. Consumo ≥ 300: tarifa 0.65, cargo 12
  3. Consumo ≥ 100: tarifa 0.45, cargo 8
  4. caso por defecto (< 100): tarifa 0.30, cargo 5

Después calculo total = consumo × tarifa + cargo (fuera de los SI, una sola vez).

🏷️ Leyenda

  • consumo = KWh consumidos (entrada, Real)
  • tarifa = tarifa por KWh según rango (intermedia, Real)
  • cargo = cargo fijo según rango (intermedia, Real)
  • total = monto total a pagar (salida, Real)

🪜 Pseudocódigo

INICIO
    Real: consumo, tarifa, cargo, total
    Leer consumo
    SI consumo >= 600
        tarifa = 0.80
        cargo = 18
    SINO
        SI consumo >= 300
            tarifa = 0.65
            cargo = 12
        SINO
            SI consumo >= 100
                tarifa = 0.45
                cargo = 8
            SINO
                tarifa = 0.30
                cargo = 5
            FIN_SI
        FIN_SI
    FIN_SI
    total = consumo * tarifa + cargo
    Mostrar "Monto total: S/. ", total
FIN

Solución · Año bisiesto (3 pts)

🎯 ¿Qué nos pidieron?

Aplicar la regla del calendario gregoriano: un año es bisiesto si (div 4 AND no div 100) OR (div 400). Es un caso de SI-SINO simple con una condición lógica compuesta.

🧠 Razonamiento

La condición clave es:

((año MOD 4 = 0) AND (año MOD 100 ≠ 0)) OR (año MOD 400 = 0)

Recordatorio: "divisible entre X" se traduce como año MOD X = 0; "NO divisible entre X" se traduce como año MOD X ≠ 0 (que en este curso se escribe <>).

🏷️ Leyenda

  • año = año a verificar (entrada, Entero)

🪜 Pseudocódigo

INICIO
    Entero: año
    Leer año
    SI ((año MOD 4 = 0) AND (año MOD 100 <> 0)) OR (año MOD 400 = 0)
        Mostrar "el año ", año, " es bisiesto"
    SINO
        Mostrar "el año ", año, " NO es bisiesto"
    FIN_SI
FIN
💡 Versión alternativa con SI anidado

Si te parece confusa la condición compuesta, también puedes hacerlo con SI anidados:

SI año MOD 400 = 0
    // caso especial: divisible entre 400 → bisiesto
    bisiesto
SINO
    SI año MOD 100 = 0
        // divisible entre 100 (pero no 400) → NO bisiesto
        no bisiesto
    SINO
        SI año MOD 4 = 0
            bisiesto
        SINO
            no bisiesto
        FIN_SI
    FIN_SI
FIN_SI

Ambas versiones son equivalentes y dan el puntaje completo. La primera es más concisa, la segunda más explícita.

Solución · N notas: aprobados, desaprobados y promedio (4 pts)

🎯 ¿Qué nos pidieron?

Procesar N notas y obtener 3 resultados simultáneamente: cantidad de aprobados, cantidad de desaprobados y promedio general. Es el patrón clásico PARA + SI con 2 contadores + 1 acumulador.

🧠 Razonamiento

Como conozco la cantidad de iteraciones (N), uso PARA. Dentro del ciclo:

  • Acumulo cada nota en suma (para luego dividir y obtener el promedio).
  • Pregunto si la nota es ≥ 10.5: si sí, incremento aprob; si no, incremento desap.

Después del PARA calculo promedio = suma / N. Nota: el promedio es Real, así que suma también debe ser Real (las notas tienen decimales).

🏷️ Leyenda

  • N = cantidad de alumnos (entrada, Entero)
  • nota = nota leída en cada iteración (Real)
  • i = control del PARA (Entero)
  • aprob = contador de aprobados (salida, Entero)
  • desap = contador de desaprobados (salida, Entero)
  • suma = acumulador de todas las notas (intermedia, Real)
  • promedio = promedio general (salida, Real)

🪜 Pseudocódigo

INICIO
    Entero: N, i, aprob, desap
    Real: nota, suma, promedio

    Leer N
    aprob = 0
    desap = 0
    suma = 0

    PARA i = 1 HASTA N HACER
        Leer nota
        suma = suma + nota
        SI nota >= 10.5
            aprob = aprob + 1
        SINO
            desap = desap + 1
        FIN_SI
    FIN_PARA

    promedio = suma / N

    Mostrar "Aprobados: ", aprob
    Mostrar "Desaprobados: ", desap
    Mostrar "Promedio: ", promedio
FIN

Solución · Suma de cifras (3 pts)

🎯 ¿Qué nos pidieron?

Calcular la suma de los dígitos de un número, sin saber cuántas cifras tiene. Es el patrón de extracción de cifras con MIENTRAS + DIV/MOD que vimos en Semana 6.

🧠 Razonamiento

El truco genérico: extraer la última cifra con MOD 10, sumarla, y eliminar la última cifra con DIV 10. Repetir hasta que el número se haga 0.

  • Como puedo modificar el número original (lo voy reduciendo), uso una copia auxiliar para no perderlo (en este ejercicio no lo pide, pero es buena práctica).
  • Inicializo suma = 0 (acumulador de sumas).
  • El MIENTRAS continúa mientras num > 0: cuando llegue a 0, ya extraí todas las cifras.

🏷️ Leyenda

  • num = número original ingresado (entrada, Entero)
  • aux = copia auxiliar para iterar sin perder el original (Entero)
  • cifra = cifra extraída en cada iteración (Entero)
  • suma = acumulador de cifras (salida, Entero)

🪜 Pseudocódigo

INICIO
    Entero: num, aux, cifra, suma

    Leer num
    aux = num      // guardo copia para no perder el original
    suma = 0

    MIENTRAS aux > 0 HACER
        cifra = aux MOD 10     // última cifra
        suma = suma + cifra
        aux = aux DIV 10     // elimino la última cifra
    FIN_MIENTRAS

    Mostrar "Suma de cifras de ", num, ": ", suma
FIN

6

Solución · Promedio de positivos hasta sentinela (3 pts)

🎯 ¿Qué nos pidieron?

Leer números hasta que ingresen un negativo, calcular el promedio de los positivos. Es el patrón clásico de priming read + MIENTRAS con contador y acumulador, más un detalle: manejar el caso de división por cero si el primer número ya es negativo.

🧠 Razonamiento

Estructura del patrón:

  1. Inicializo cant = 0 (contará positivos) y suma = 0 (acumulará positivos).
  2. Priming read: leo n ANTES del MIENTRAS.
  3. MIENTRAS n ≥ 0: acumulo, incremento contador, leo el siguiente n.
  4. Después del ciclo: pregunto si cant > 0 (al menos un positivo). Si sí, calculo y muestro promedio. Si no, muestro mensaje de error.

¿Por qué el SI al final? Para evitar dividir entre cero. Si el primer número leído ya es negativo, el MIENTRAS ni siquiera se ejecuta y cant queda en 0. Sin el SI, calcularíamos suma / 0 → error.

🏷️ Leyenda

  • n = número leído en cada iteración (Real)
  • cant = contador de positivos leídos (Entero)
  • suma = acumulador de positivos (Real)
  • promedio = promedio calculado (salida, Real)

🪜 Pseudocódigo

INICIO
    Entero: cant
    Real: n, suma, promedio

    cant = 0
    suma = 0

    Leer n                  // PRIMING READ
    MIENTRAS n >= 0 HACER
        suma = suma + n
        cant = cant + 1
        Leer n              // LECTURA SUBSIGUIENTE
    FIN_MIENTRAS

    SI cant > 0
        promedio = suma / cant
        Mostrar "Promedio: ", promedio
    SINO
        Mostrar "No se ingresaron números positivos"
    FIN_SI
FIN
⚠️ Detalle sutil: ¿n >= 0 o n > 0?

El enunciado dice "termina cuando ingrese un negativo". Eso implica que el cero (0) NO es la señal de parada, así que el cero debería procesarse. Pero el cero no es positivo (es cero). Como el promedio "de positivos" estrictamente excluye el 0, hay dos interpretaciones válidas:

  • Interpretación amplia (más común): tratar el 0 como "no negativo" → entra al ciclo pero suma 0. Condición: n >= 0.
  • Interpretación estricta: solo positivos verdaderos. Condición: n > 0 → el 0 también para el ciclo.

En el examen real, ambas se aceptan si está justificada en la leyenda. Yo escogí la primera porque "termina cuando ingrese un negativo" es la regla literal del enunciado.
