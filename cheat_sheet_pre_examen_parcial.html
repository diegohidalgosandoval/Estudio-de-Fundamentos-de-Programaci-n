Pseudocódigo 101 · Sintaxis del curso

📐 Estructura mínima de cualquier algoritmo

INICIO
    // 1. Declaración de variables (con sus tipos)
    Tipo: var1, var2, ...

    // 2. Entrada de datos
    Leer var1, var2

    // 3. Procesamiento (asignaciones, decisiones, ciclos)
    resultado = var1 + var2

    // 4. Salida
    Mostrar resultado
FIN
⚠️ NO OLVIDES (en cada ejercicio del examen)

Es obligatorio incluir una leyenda de variables (qué representa cada nombre). Sin leyenda → puntos perdidos automáticamente.

🏷️ Tipos de datos

TipoQué guardaEjemplos
EnteroNúmeros sin decimales5, -3, 0, 100
RealNúmeros con decimales3.14, -0.5, 10.0
CarácterUna sola letra/símbolo'a', 'X', '7'
CadenaTexto (varios caracteres)"hola", "aprobado"
LógicoVerdadero o falsoV, F

📋 Palabras clave reservadas

INICIO FIN Leer Mostrar SI SINO FIN_SI MIENTRAS HACER FIN_MIENTRAS REPETIR HASTA QUE PARA FIN_PARA

📝 Convenciones del curso

✅ Hacer

  • = para asignación y comparación
  • INICIO/FIN en mayúsculas
  • Leer/Mostrar con primera mayúscula
  • Indentar 4 espacios dentro de SI, MIENTRAS, etc.
  • Comentarios con //
  • Cadenas con comillas: "texto"

⚠️ Errores típicos

  • Olvidar declarar tipos antes de usar
  • Olvidar FIN_SI / FIN_MIENTRAS / FIN_PARA
  • Mal indentado (puntos perdidos)
  • Confundir = con := o ←
  • Usar tipo Entero cuando hay decimales
  • Olvidar la leyenda de variables

🧮 Aritméticos

⚖️ Relacionales (devuelven V o F)

> mayor que < menor que >= mayor o igual <= menor o igual = igual <> distinto

🔣 Lógicos

🔗 AND (Y)

V solo si las DOS son V. Exigente.

(edad >= 18) AND (DNI = V)

🔀 OR (O)

V si al menos una es V. Permisivo.

(dia = "sab") OR (dia = "dom")

🔄 NOT (NO)

Invierte: V→F, F→V.

NOT (n > 0) = n <= 0

🏆 Jerarquía completa (de mayor a menor prioridad)

  1. ( )Paréntesis (siempre primero)
  2. ^Potencia
  3. NOTNegación lógica
  4. * / DIV MOD ANDMultiplicación, división y AND
  5. + − ORSuma, resta y OR
  6. < > <= >= = <>Relacionales (al final)
⚠️ Regla práctica

Cuando combines relacionales y lógicos, SIEMPRE usa paréntesis alrededor de cada comparación. Te ahorra errores y hace el código más legible.

❌ edad >= 18 AND DNI = V    ✅ (edad >= 18) AND (DNI = V)

🧮 Leyes de De Morgan (para invertir condiciones)

📐 Las dos leyes
NOT (a AND b) ≡ (NOT a) OR (NOT b)
NOT (a OR b) ≡ (NOT a) AND (NOT b)

El AND se vuelve OR (y viceversa), y cada parte se niega.

🔁 Aplicación: convertir MIENTRAS ↔ REPETIR

// MIENTRAS continúa cuando V
MIENTRAS (n >= 1) AND (n <= 12) HACER
    ...
FIN_MIENTRAS
// REPETIR sale cuando V (negar la cond)
REPETIR
    ...
HASTA QUE (n < 1) OR (n > 12)

Decisiones: SI / SI-SINO / SI anidado

📌 SI simple

Solo UN caso requiere acción.

SI condición
    // se ejecuta solo si V
    bloque_P
FIN_SI

📌 SI-SINO

AMBOS casos requieren acción distinta.

SI condición
    bloque_V
SINO
    bloque_F
FIN_SI

🪜 SI anidado (3+ casos): patrón "filtro en cascada"

SI condición1            // la MÁS restrictiva
    bloque_P
SINO
    SI condición2
        bloque_Q
    SINO
        SI condición3
            bloque_R
        SINO
            bloque_S         // caso por defecto
        FIN_SI
    FIN_SI
FIN_SI
🎯 Regla de oro de la anidación

Pon primero la condición MÁS restrictiva. Si pones primero la más amplia, los casos restrictivos nunca se ejecutan.

Ejemplo: para descuentos por cantidad, primero n > 50 (10%), luego n >= 20 (6%), luego sin descuento.

🌍 Traducciones más comunes (español → pseudocódigo)

Frase en españolPseudocódigo
"mayor o igual a 10.5"x >= 10.5
"5 o más", "al menos 5"x >= 5
"más de 50" (estricto)x > 50
"inferior a 2000" (estricto)x < 2000
"entre 7 y 9" (incluye ambos)(x >= 7) AND (x <= 9)
"[12 a 16)" (cerrado-abierto)(x >= 12) AND (x < 16)
"sábado o domingo"(d = "sab") OR (d = "dom")
"NO es positivo"x <= 0

Repetición: MIENTRAS / REPETIR-HASTA / PARA

📊 Cuadro comparativo (memorízalo)

CaracterísticaMIENTRASREPETIR-HASTAPARA

🪜 Sintaxis lado a lado

// MIENTRAS
MIENTRAS condición HACER
    cuerpo
    actualización
FIN_MIENTRAS
// REPETIR-HASTA
REPETIR
    cuerpo
    actualización

🧭 Árbol de decisión (¿cuál uso?)

🌲 Pregúntate en este orden
  1. ¿Conozco EXACTAMENTE cuántas iteraciones? (ej: "los 30 alumnos") → PARA.
  2. ¿Quiero ejecutar al menos UNA vez y revisar después? (ej: validar entrada) → REPETIR-HASTA.
  3. En cualquier otro caso (cantidad desconocida, depende del usuario o datos) → MIENTRAS.

📋 Las 4 partes obligatorias de TODO ciclo

1️⃣ Inicialización

Variables de control y acumuladores (antes del ciclo).

cont = 0
suma = 0

2️⃣ Condición de parada

Una expresión lógica que algún día cambiará y permitirá salir.

MIENTRAS i <= 10 HACER

3️⃣ Cuerpo del ciclo

Las instrucciones que se repiten.

Leer x
suma = suma + x

4️⃣ Actualización

Cambiar las variables de la condición. Sin esto → bucle infinito.

i = i + 1
🔥 Bucle infinito

El error más temido. Olvidaste actualizar la variable de la condición → el ciclo nunca termina. SIEMPRE asegura que algo dentro del cuerpo modifique las variables que pregunta la condición.

6

Patrones esenciales (memorízalos como plantillas)

🔢 Contador vs Acumulador (la diferencia clave)

🔢 Contador

Suma una cantidad CONSTANTE (típicamente 1) cada iteración. Cuenta CUÁNTAS veces ocurre algo.

cont = cont + 1

Inicializa en: 0

➕ Acumulador (totalizador)

Suma valores VARIABLES (los que se leen). Calcula la SUMA total.

suma = suma + N

Inicializa en: 0 (suma) o 1 (producto)

⚠️ ¡CRUCIAL!

Acumulador de PRODUCTO va inicializado en 1, NO en 0. Si pones 0, el primer producto da 0 y todo se vuelve 0. El 1 es la identidad del producto: 1 × x = x.

📚 Las 6 plantillas que SIEMPRE debes recordar

1. CONTAR cuántos cumplen X (PARA + SI + contador)

cont = 0
PARA i = 1 HASTA N HACER
    Leer x
    SI condición(x)        // ej: x MOD 2 = 0 (es par)
        cont = cont + 1
    FIN_SI
FIN_PARA

2. SUMAR todos / calcular PROMEDIO

suma = 0
PARA i = 1 HASTA N HACER
    Leer x
    suma = suma + x
FIN_PARA
promedio = suma / N

3. MÁXIMO y MÍNIMO (inicializa con 1er elemento, NUNCA con 0)

Leer x
max = x
min = x
PARA i = 2 HASTA N HACER      // arranco en 2 (el 1 ya lo leí)
    Leer x
    SI x > max
        max = x
    FIN_SI
    SI x < min
        min = x
    FIN_SI
FIN_PARA

4. LECTURA HASTA SENTINELA (priming read + MIENTRAS)

cont = 0
Leer n                  // PRIMING READ (1ª lectura, fuera del ciclo)
MIENTRAS n > 0 HACER      // hasta que ingrese un negativo
    cont = cont + 1
    Leer n              // LECTURA SUBSIGUIENTE (dentro)
FIN_MIENTRAS

5. BÚSQUEDA con bandera (flag)

encontrado = F
i = 1
MIENTRAS (i <= N) AND (encontrado = F) HACER
    SI V[i] = buscado
        encontrado = V
    FIN_SI
    i = i + 1
FIN_MIENTRAS

6. EXTRAER CIFRAS de un número (DIV/MOD + MIENTRAS)

MIENTRAS num > 0 HACER
    cifra = num MOD 10      // extraigo la última cifra
    // procesar la cifra (sumarla, multiplicarla, etc.)
    num = num DIV 10       // elimino la última cifra
FIN_MIENTRAS

📌 Reglas de oro

✓ Lo que NUNCA debe faltar
  • Etiquetas V/F (o Sí/No) en cada salida del rombo de decisión.
  • Las flechas tienen cabeza visible (no son solo líneas).
  • Solo el rombo tiene 2 salidas. Los demás tienen 1.
  • Inicio y Fin son únicos en cada algoritmo.
  • Las flechas fluyen de arriba hacia abajo (lectura natural).

🔁 Diagrama mental de cada estructura

SI / SI-SINO

Rombo → si V, va a un proceso; si F, salta o va a otro proceso. Las dos ramas se reúnen después.

MIENTRAS

Rombo PRIMERO → si V, ejecuta proceso → vuelve al rombo (forma un lazo). Si F, sale.

REPETIR-HASTA

Proceso PRIMERO → rombo → si F, vuelve al proceso. Si V, sale. El lazo es por la rama F.

DIV y MOD · Manipulación numérica

🧮 Identidad fundamental
a = b × (a DIV b) + (a MOD b)

Verificación: 17 = 5 × 3 + 2 ✓

📊 Tabla de referencia rápida

OperaciónDIV (cociente)MOD (resto)
17 / 532
75 / 893
99 / 1569
154 / 121210
847 / 10847
8 / 1008

🎯 Las 5 aplicaciones más útiles de MOD

1️⃣ Detectar paridad

n MOD 2 = 0 → par
n MOD 2 = 1 → impar

2️⃣ Detectar divisibilidad

a MOD b = 0 → b divide a a
Ej: bisiesto → año MOD 4 = 0

3️⃣ Última cifra

n MOD 10 = última cifra
Ej: 847 MOD 10 = 7

4️⃣ Eliminar última cifra

n DIV 10 = sin última cifra
Ej: 847 DIV 10 = 84

5️⃣ Descomposición de número de 3 cifras

c = num DIV 100
d = (num MOD 100) DIV 10
u = num MOD 10

Para 4 cifras: añade m = num DIV 1000 y ajusta los demás.

💡 Patrón "invertir un número" (cualquier cantidad de cifras)

inv = 0
MIENTRAS num > 0 HACER
    cifra = num MOD 10
    inv = inv * 10 + cifra     // "empuja" cifras a la izquierda
    num = num DIV 10
FIN_MIENTRAS
// 1234 → 4321

Plantillas de problemas típicos del examen

Reconoce el tipo de problema y aplica la plantilla. Estos son los 5 tipos que aparecen una y otra vez.

🎯 Tipo 1: "Descuento/comisión por rangos"

Identifícalo: hay 3+ rangos excluyentes (ventas, kilos, horas, etc.) con tarifas distintas.

Patrón: SI anidado en cascada, de mayor a menor restrictivo.

SI x >= UMBRAL_ALTO
    aplicar tarifa alta
SINO
    SI x >= UMBRAL_MEDIO
        aplicar tarifa media
    SINO
        SI x >= UMBRAL_BAJO
            aplicar tarifa baja
        SINO
            sin tarifa / por defecto
        FIN_SI
    FIN_SI
FIN_SI

🏆 Tipo 2: "Mayor / Menor de N números"

Identifícalo: "determine el mayor", "el menor", "el más alto", de N valores.

Patrón: inicializa max/min con el 1er elemento, compara desde el 2°.

Leer N
Leer x
mayor = x
PARA i = 2 HASTA N HACER
    Leer x
    SI x > mayor
        mayor = x
    FIN_SI
FIN_PARA

🔢 Tipo 3: "Contar X en N elementos"

Identifícalo: "cuántos pares hay", "cuántas 'a'", "cuántos aprobados".

Patrón: PARA + SI + contador.

cont = 0
PARA i = 1 HASTA N HACER
    Leer x
    SI condición(x)
        cont = cont + 1
    FIN_SI
FIN_PARA

🔁 Tipo 4: "Procesar hasta sentinela"

Identifícalo: "hasta que ingrese un negativo", "hasta que escriba 0", cantidad desconocida.

Patrón: MIENTRAS con priming read.

Leer n                // PRIMING READ
MIENTRAS n >= 0 HACER
    procesar n
    Leer n            // siguiente lectura
FIN_MIENTRAS

🎯 Tipo 5: "Mayor de 3 números (A, B, C)"

Identifícalo: "el mayor de tres números diferentes".

Patrón: SI anidado de 2 niveles.

SI A > B
    SI A > C
        mayor = A
    SINO
        mayor = C
    FIN_SI
SINO
    SI B > C
        mayor = B
    SINO
        mayor = C
    FIN_SI
FIN_SI

🔥 Top 15 errores que cuestan puntos en el examen

⚠️ Lee esta sección 3 veces antes del examen

Estos son los errores más comunes que el profesor descuenta. Si los evitas, ganas puntos automáticos.

❌ 1. Olvidar la leyenda de variables

Es obligatorio. Lista qué representa cada variable antes del pseudocódigo.

❌ 2. Orden incorrecto en SI anidado

Pones primero la cond. amplia → los casos restrictivos nunca corren. Siempre de mayor a menor.

❌ 3. Confundir "más de" con "o más"

"Más de 50" = > 50 (NO incluye 50). "50 o más" = >= 50 (SÍ incluye 50).

❌ 4. FIN_SI / FIN_MIENTRAS faltantes

Cuenta SI ↔ FIN_SI: deben coincidir 1 a 1. Igual para MIENTRAS y PARA.

❌ 5. No inicializar contadores

cont = 0 ANTES del ciclo. Si no, el cálculo parte de "basura".

❌ 6. Acumulador de producto en 0

Producto va en 1 (identidad). En 0, todo se vuelve 0.

❌ 7. max/min inicializado en 0

Falla con valores negativos. Inicializa con el 1er elemento.

❌ 8. Bucle infinito (olvido actualizar)

SIEMPRE asegura que algo del cuerpo cambie las variables de la condición.

❌ 9. Diagrama sin etiquetas V/F

Cada salida del rombo debe estar marcada con V (Sí) o F (No).

❌ 10. Símbolos incorrectos en diagrama

Inicio/Fin = óvalo. E/S = paralelogramo. Proceso = rectángulo. Decisión = rombo.

❌ 11. Mal indentado

Indenta 4 espacios dentro de SI, MIENTRAS, etc. Hace el código legible y se valora.

❌ 12. Olvidar priming read

En MIENTRAS donde la cond. depende del usuario, lee fuera ANTES y dentro DESPUÉS.

❌ 13. Usar tipo Entero para decimales

Promedios, precios, sueldos → Real. Cantidades discretas → Entero.

❌ 14. Confundir índice con valor

i es la posición; V[i] es el valor en esa posición. No los mezcles.

❌ 15. No probar caso límite

Si la cond. es ≥ 10.5, prueba con 10.5 EXACTO. Es donde más fallan los algoritmos.

Checklist final · Domínalos antes del examen

🎯 Estrategia el día del examen
  1. Lee toda la prueba primero (5 min) para identificar dificultad.
  2. Empieza por el ejercicio que más dominas. Gana confianza.
  3. En cada ejercicio: leyenda → pseudocódigo → diagrama. NO inviertas el orden.
  4. Usa lápiz para borrador, pero entrega en lapicero (sin enmendaduras).
  5. Si te quedas atascado, pasa al siguiente y vuelve después.
  6. Reserva los últimos 10 min para revisar trazas y verificar cierres FIN_SI/FIN_MIENTRAS.
  7. Respira. Si dominas las plantillas, los problemas son variaciones de lo mismo.
