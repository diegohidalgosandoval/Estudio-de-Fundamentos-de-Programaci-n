Tema 1 · break y continue

La Semana 13 introduce dos instrucciones que dan control quirúrgico sobre el flujo de un bucle: break para salir abruptamente, y continue para saltarse una iteración específica. También veremos cómo construir menús interactivos que combinan estas herramientas con el patrón while True + break.

1.1 · La instrucción break

💡
Intuición

Imagina que estás buscando las llaves de tu casa en una caja de objetos varios. En cuanto las encuentras, dejas de buscar — no tiene sentido revisar el resto de la caja. Eso es exactamente break: en cuanto se cumple una condición, el bucle se detiene de inmediato, sin completar las iteraciones que faltaban.

Sin break, el bucle seguiría revisando todos los objetos restantes aunque ya hayas encontrado lo que buscabas — un desperdicio de tiempo y esfuerzo computacional.

Definición formal

La instrucción break es una instrucción de salto que, cuando se ejecuta dentro de un bucle (for o while), termina inmediatamente ese bucle y transfiere el control a la primera instrucción que está fuera del bloque del bucle.

🔑
Regla fundamental

break interrumpe el bucle más interno que lo contiene. Si hay bucles anidados, solo termina el más cercano. El bucle exterior sigue ejecutándose con normalidad.

¿Por qué existe break? ¿Qué problema vino a resolver?

Antes de break, si querías salir de un bucle antes de que terminara su condición natural, tenías que usar una variable bandera (flag) extra: una variable booleana que se activaba al encontrar la condición y se chequeaba en la condición del while. Funciona, pero es más código y más difícil de leer.

break hace que la intención sea inmediatamente legible: «en cuanto pase X, sal del bucle». Sin variables auxiliares, sin condiciones compuestas.

Mini-ejemplo 3 — break dentro de un while

break funciona exactamente igual dentro de un while. Este es un uso muy común:

código Python — break en while
# Pedir números hasta que el usuario ingrese 0
suma = 0
while True:                       # bucle infinito intencional
    n = int(input("Número (0=salir): "))
    if n == 0:
        break                       # sale del while True
    suma += n
print("Suma:", suma)               # aquí llega tras el break
5 3 7 0
🔎
El patrón while True + break

while True: crea un bucle infinito — la condición siempre es verdadera, así que el bucle nunca terminaría por sí solo. El único mecanismo de salida es un break interno. Este patrón es deliberado y muy útil cuando no se sabe de antemano cuántas iteraciones se necesitan, y la condición de salida se evalúa dentro del cuerpo del bucle. Lo veremos extensamente en el Tema 2 de esta semana (menús).

Heads-up de examen · break

1. Orden relativo del break y el print:
El examen puede preguntarte qué imprime un código donde el print y el break están en diferente orden. La regla es: lo que está antes del break sí ejecuta; lo que está después del break en el mismo bloque, no.

2. ¿Qué se ejecuta después del break?
Solo la primera instrucción fuera del bucle. Todo lo que está dentro del bucle (mismo nivel de indentación) se omite.

3. Break en bucles anidados:
Si te preguntan qué pasa con un break en un for dentro de otro for, recuerda: solo sale del más interno.

4. El código muerto num = 0:
Si el profe usa el estilo num=0; for num in range(...):, no te confundas — la asignación inicial no tiene efecto. El for empieza siempre desde el primer valor del range.

1.2 · La instrucción continue

💡
Intuición

Imagina que estás revisando una lista de tareas pendientes. Llegas a una tarea que no puedes hacer hoy (por ejemplo, necesitas esperar que te llamen). En lugar de quedarte bloqueado, la saltas y pasas a la siguiente tarea de la lista. No abandonas la lista entera — simplemente omites esa tarea concreta.

Eso es continue: salta la iteración actual y vuelve al inicio del bucle para continuar con la siguiente. El bucle no termina — solo esa iteración queda incompleta.

Definición formal

La instrucción continue es una instrucción de salto que, cuando se ejecuta dentro de un bucle, abandona el resto de las instrucciones de la iteración actual y transfiere el control de vuelta al inicio del bucle para evaluar la condición de la siguiente iteración. El bucle en sí continúa ejecutándose.

🔑
Diferencia esencial con break

break termina el bucle completo — las iteraciones restantes nunca ocurren.
continue termina solo la iteración actual — el bucle sigue con la siguiente iteración normalmente.

¿Por qué existe continue? ¿Qué problema vino a resolver?

Sin continue, para "saltarse" ciertos casos dentro de un bucle necesitarías envolver todo el cuerpo restante del bucle dentro de un bloque if. Con continue, la intención es más clara: dices explícitamente «para este caso, no hagas nada más en esta iteración».

Compara las dos formas equivalentes:

Sin continue (menos legible)

código Python
for i in range(10):
    if i != 5:          # condición negada
        print(i)
        # ... más instrucciones
        # ... todo anidado un nivel más

Con continue (más legible)

código Python
for i in range(10):
    if i == 5:
        continue          # caso especial al inicio
    print(i)
    # ... más instrucciones al mismo nivel

La versión con continue es preferible porque maneja el caso especial al principio del bucle como una "guardia" (guard clause), y el resto del código queda al nivel normal de indentación, sin anidarse innecesariamente.

Mini-ejemplo 3 — continue en while y la trampa del bucle infinito

⚠️
Trampa crítica: continue + while + contador

En un while, si el incremento del contador está después del continue, cuando la condición se cumpla el continue saltará el incremento, el contador nunca avanzará, y el bucle se vuelve infinito.

❌ Incorrecto — bucle infinito

código Python — PELIGROSO
i = 0
while i < 5:
    if i == 2:
        continue    # i nunca llega a i+=1
    print(i)
    i += 1      # ← NUNCA ejecuta cuando i==2

Cuando i == 2, el continue salta el i += 1. El while evalúa de nuevo: ¿2 < 5? Sí. Vuelve a i == 2. continue. Para siempre.

✓ Correcto — incremento antes del continue

código Python — seguro
i = 0
while i < 5:
    i += 1      # ← incremento PRIMERO
    if i == 2:
        continue    # i ya fue incrementado
    print(i)

Ahora cuando i == 2 el continue salta el print, pero i ya fue incrementado. El while ve 3 < 5, sigue. Salida: 1, 3, 4, 5.

código Python — continue en while (versión correcta ejecutable)
# Imprimir 1..7 saltando los pares
i = 0
while i < 7:
    i += 1             # incremento ANTES del continue
    if i % 2 == 0:    # si es par
        continue       # salta el print para este par
    print(i)           # solo impares llegan aquí

🔎
Interpretación de la salida

Con break: 0 1 2 3 4 — el bucle ejecutó 5 iteraciones y paró. Las iteraciones 6, 7, 8, 9, 10 (i=5..9) nunca existieron.

Con continue: 0 1 2 3 4 6 7 8 9 — el bucle ejecutó las 10 iteraciones. La de i=5 existió pero no imprimió nada. Las demás funcionaron con normalidad.

Observa el contraste: con break el 5 nunca aparece y el bucle termina ahí. Con continue el 5 tampoco aparece pero el bucle sigue hasta el final.

🎯
Heads-up de examen · continue

1. ¿El bucle termina o sigue?
Si el enunciado dice «salta ciertos valores pero procesa todos los demás», es continue. Si dice «detente cuando encuentres X», es break.

2. La trampa del while + continue + contador:
En un examen pueden darte un while con continue y el incremento después. Debes detectar el bucle infinito. La solución: incremento siempre antes del continue.

3. El error del PDF («del 6 al 10»):
range(10) va de 0 a 9. Si en el examen aparece un ejercicio similar con range(10), el último valor es siempre 9. Para llegar al 10 se necesita range(11).

4. Equivalencia con negación del if:
if cond: continue; [resto] es equivalente a if not cond: [resto]. Ambas formas pueden aparecer en el examen.

1
Simulación de dado — contar intentos hasta sacar el 5 break

Elabore un programa que simule «n» lanzamientos de un dado y determine la cantidad de intentos realizados hasta que salió el número 5. (utilizar BREAK)

a · Comprensión del problema

Se simula tirar un dado de 6 caras hasta n veces. El dado puede caer en cualquier valor del 1 al 6 (equiprobable). En cuanto salga un 5, se detiene el proceso y se reporta cuántos lanzamientos fueron necesarios.

  • Entrada: n (número máximo de lanzamientos)
  • Proceso: generar caras aleatorias, contar hasta que sale 5 o se agota n
  • Salida: número de intento en que salió el 5 (o aviso de que no salió)

Casos borde:

  • n = 1: solo hay un lanzamiento; puede o no ser 5
  • n = 0: no hay lanzamientos; el 5 nunca aparece
  • El 5 sale en el primer intento
  • El 5 nunca sale dentro de los n intentos (necesitamos manejar este caso)

b · Leyenda de variables

c · Estrategia

Usamos un for que va de 0 a n−1 (un intento por iteración). Dentro, generamos un número aleatorio entre 1 y 6. Si es 5, reportamos el número de intento (i+1 porque el índice empieza en 0) y hacemos break para no seguir lanzando.

¿Por qué for y no while? Porque tenemos un límite fijo de intentos (n). El for garantiza que no superemos ese límite. El break nos saca antes si encontramos el 5.

Para manejar el caso borde «el 5 no apareció», usamos una variable bandera encontrado.

Tema 2 · Programas con Menú de Usuario

Un menú de usuario es una lista de opciones numeradas que el programa muestra para que el usuario elija qué acción ejecutar. Es la forma más natural de hacer programas interactivos de consola. Esta semana aprendemos a construirlos desde el patrón más simple hasta el patrón completo con repetición indefinida.

2.1 · Concepto y estructura básica

💡
Intuición

Imagina un cajero automático: te muestra una pantalla con opciones («1. Retirar dinero», «2. Ver saldo», «3. Salir»), tú presionas un número, y el cajero ejecuta la acción correspondiente. Eso es exactamente la estructura de un programa con menú.

Sin menú, tendrías que reiniciar el programa cada vez que quieras hacer otra operación. El menú convierte un programa de un solo uso en una herramienta reutilizable dentro de una misma sesión.

Los tres pasos del ciclo de un menú

Todo programa con menú, sin importar cuán complejo sea, sigue este ciclo de tres pasos:

PasoQué hace el programaQué hace el usuario
1 · Mostrar Imprime las opciones disponibles con sus números Lee las opciones
2 · Leer Espera que el usuario ingrese un número Escribe el número de la opción deseada
3 · Ejecutar Identifica la opción y realiza la acción correspondiente Ve el resultado de su elección

Versión mínima: menú de ejecución única

La forma más simple de un menú se ejecuta una sola vez: muestra las opciones, lee la elección, ejecuta la acción y termina. No hay repetición.

🔑
Por qué int(input(...))

input() siempre devuelve una cadena de texto (str), no un número. Si el usuario escribe 1, Python recibe el string "1". Al comparar con if opcion == 1, estaríamos comparando un string con un entero, lo que en Python siempre da False.

La solución es convertir con int(): int(input(...)) convierte el string ingresado a entero antes de guardarlo en opcion.

2.2 · El patrón while True + break

💡
Intuición — el cajero que no se apaga solo

El cajero automático no termina después de una operación. Vuelve al menú principal y espera la siguiente elección. Solo se "apaga" cuando el usuario elige «Salir». Eso requiere que el ciclo de tres pasos se repita indefinidamente hasta que el usuario decida parar.

En programación, cuando no sabemos de antemano cuántas veces el usuario querrá usar el menú, usamos un bucle que se repite para siempre (while True) con una salida controlada (break) cuando el usuario elige la opción de salir.

El problema: ¿cuántas veces se repite?

No podemos usar un for porque no sabemos cuántas veces el usuario va a interactuar con el menú — puede elegir 1 opción o 50. Un while con condición sobre la opción también funciona, pero tiene un problema elegante: necesitamos leer la opción antes de entrar al while (para evaluar la condición) y también dentro del while (para la siguiente iteración). Eso genera código duplicado.

La solución elegante es while True + break:

❌ Con while y condición (código duplicado)

código Python
opcion = int(input("Opción: "))  # 1ª lectura
while opcion != 3:
    # ... procesar ...
    opcion = int(input("Opción: "))  # 2ª lectura

El input() aparece dos veces. Si cambia el mensaje, hay que cambiarlo en dos lugares — propenso a errores.

✓ Con while True + break (limpio)

código Python
while True:
    # mostrar menú
    opcion = int(input("Opción: "))
    if opcion == 3:
        break         # única salida
    # ... procesar opciones 1 y 2 ...

El input() aparece solo una vez, dentro del bucle. El break es la única salida. Más limpio y mantenible.

🎯
Heads-up de examen · Menús

1. int(input()) es obligatorio:
Si usas solo input() sin int(), la comparación opcion == 1 siempre dará False porque estarás comparando el string "1" con el entero 1. Es el error más frecuente.

2. El break va DENTRO del elif, no después:
El break debe estar dentro del bloque elif opcion == 3:, no después del if/elif/else. Si lo pones fuera, el bucle siempre termina tras la primera iteración.

3. Manejar opción inválida:
El profe no siempre lo pide explícitamente, pero si el enunciado dice «el programa solicita repetidamente», implica que una opción fuera del rango debe volver al menú, no romper el programa. Incluye siempre un else.

4. while True vs while opcion != N:
Ambos funcionan. El profe usa while True + break. Si en el examen usas la variante con condición, asegúrate de leer la opción antes del while (para que la condición se pueda evaluar) y también dentro.

Conversión de temperatura °F ↔ °C con menú repetido while True + break

Elabore un programa que convierta un valor de temperatura entre °F y °C. La fórmula es: C = 5/9(F−32). El programa debe solicitar ingreso de opciones en forma repetida y solamente finalizará cuando se ingrese el número 3. (PDF p. 14)

a · Comprensión y fórmulas

El programa ofrece dos conversiones:

  • Opción 1: °F → °C usando \( C = \dfrac{5}{9}(F - 32) \)
  • Opción 2: °C → °F usando \( F = \dfrac{9}{5}C + 32 \)
  • Opción 3: Salir
📐
Las fórmulas de temperatura — de dónde vienen

Fahrenheit y Celsius son dos escalas lineales. El agua congela a 32°F y 0°C; hierve a 212°F y 100°C.

La diferencia entre los dos puntos de referencia es 180°F = 100°C, lo que da la razón \(\frac{100}{180} = \frac{5}{9}\). El desplazamiento de 32°F (origen distinto) explica el (F − 32).

Verificación rápida: \( C = \frac{5}{9}(212 - 32) = \frac{5}{9} \times 180 = 100 \) ✓
\( C = \frac{5}{9}(32 - 32) = 0 \) ✓

a · Comprensión — ¿qué hace este programa?

El programa calcula la suma de los cuadrados de los números enteros desde 1 hasta n−1 (inclusive). Es decir: \( s = 1^2 + 2^2 + 3^2 + \cdots + (n-1)^2 \)

⚠️
Detalle crítico: range(1, n) NO incluye n

range(1, n) genera los valores 1, 2, 3, …, n−1. El valor n no entra en la suma. Por ejemplo, con n=5: se suman 1²+2²+3²+4² = 1+4+9+16 = 30. El 5² = 25 no se suma.

Si el enunciado quisiera sumar hasta n inclusive, habría que usar range(1, n+1).

c · Regla general de conversión for → while

📐
Plantilla de equivalencia for ↔ while

Todo for i in range(inicio, fin) se puede convertir en un while así:

for original 
for i in range(inicio, fin):
    <cuerpo>
while equivalente 
i = inicio         # 1. inicializar
while i < fin:     # 2. condición
    <cuerpo>
    i += 1           # 3. incrementar

Los tres elementos son obligatorios: inicializar antes, condición en el while, incrementar al final del cuerpo. Si falta el incremento → bucle infinito.

Número mágico a partir de la fecha de nacimiento while anidado · DIV · MOD

En un juego se debe asignar a cada persona un número mágico que se obtiene con la siguiente regla: Se suman los dígitos de la fecha de nacimiento y se suman nuevamente los dígitos del resultado hasta obtener un solo dígito.

a · Comprensión profunda

El problema tiene dos niveles de bucle:

  • Bucle externo (while s >= 10): se repite mientras el número tenga más de un dígito, es decir, mientras sea ≥ 10. Aplica la suma de dígitos al resultado actual.
  • Bucle interno (while n > 0): extrae y suma los dígitos de un número usando las operaciones MOD (obtiene el último dígito) y DIV (elimina el último dígito).
💡
Intuición — ¿cómo se extraen los dígitos?

Imagina el número 2038 escrito verticalmente: 2 | 0 | 3 | 8. Para leer los dígitos de derecha a izquierda:

  • MOD 10 (%): da el dígito más a la derecha. 2038 % 10 = 8.
  • DIV 10 (//): elimina el dígito más a la derecha. 2038 // 10 = 203.

Repetir hasta que el número sea 0 extrae todos los dígitos de derecha a izquierda: 8, 3, 0, 2. Sumados: 8+3+0+2 = 13.

🔑
Equivalencias UDEP → Python (críticas para este ejercicio)

DIV (división entera) → // en Python
MOD (resto) → % en Python
Ejemplo: 2038 DIV 10 = 203  →  2038 // 10 = 203
Ejemplo: 2038 MOD 10 = 8    →  2038 % 10 = 8

1. ¿Cuál es la diferencia esencial entre break y continue?

break termina el bucle completo — las iteraciones restantes nunca ocurren. continue termina solo la iteración actual — el bucle sigue con la siguiente. Analogía: break es salir del examen; continue es saltar una pregunta y seguir con las demás.

2. ¿Qué imprime este código?
for i in range(10): if i == 4: break; print(i)

Imprime 0, 1, 2, 3. El break está antes del print, así que cuando i == 4, el break ejecuta antes de que print pueda mostrar el 4. El bucle termina sin imprimir el 4.

3. ¿Qué imprime este código?
for i in range(10): print(i); if i == 4: break

Imprime 0, 1, 2, 3, 4. Ahora el print está antes del if/break. Cuando i == 4, primero se imprime el 4 y luego se ejecuta el break. El 4 sí aparece. Esta diferencia de orden es la trampa más común en exámenes.

4. El PDF de S13 dice que el bucle con continue (cuando num==5) «imprime los números del 6 al 10». ¿Es correcto? Explica.

No es correcto. range(10) genera valores del 0 al 9. El valor 10 nunca se genera. El último número impreso es 9. Para llegar al 10 habría que usar range(11). Es un error tipográfico en el material oficial.

8. ¿Cuál es la trampa del continue dentro de un while cuando el incremento del contador está después del continue?

Produce un bucle infinito. Cuando se ejecuta el continue, el resto del cuerpo (incluyendo el incremento) se salta. El contador nunca avanza, la condición del while siempre es verdadera, y el programa nunca termina. La solución es poner el incremento antes del continue.
