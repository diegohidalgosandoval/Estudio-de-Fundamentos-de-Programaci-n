1

SI anidados a fondo: filtros en cascada

💡 Recordatorio

En la Semana 4 viste un avance de los SI anidados (ejercicio del triángulo). Ahora los profundizamos: son la herramienta clave cuando un problema tiene 3 o más categorías mutuamente excluyentes (descuentos por niveles, tarifas por rangos, calificaciones por letras, etc.).

1.1. ¿Qué es anidar?

Definición
SI anidado (encajado)
"Las estructuras SI interiores a otras estructuras SI se denominan anidadas o encajadas." (Joyanes). Es decir: dentro de un SI (en su rama V o F), pones otro SI completo.

1.2. Sintaxis del patrón "filtro en cascada"

El patrón más útil: cuando tienes varios rangos excluyentes, anidas SI dentro del SINO sucesivamente:

SI condición1
    // caso 1: condición1 es V
    bloque_P
SINO
    SI condición2
        // caso 2: cond1 F, cond2 V
        bloque_Q
    SINO
        SI condición3
            // caso 3: cond1 F, cond2 F, cond3 V
            bloque_R
        SINO
            // caso 4 "por defecto": ninguna se cumplió
            bloque_S
        FIN_SI
    FIN_SI
FIN_SI

1.3. Cómo "leer" un anidamiento (mentalmente)

Imagínalo como una cadena de filtros: cada SINO te lleva a una nueva pregunta, y solo entras al siguiente filtro si fallaste el anterior:

¿condición1? ─sí→ ejecuta P y SAL
   no ↓
¿condición2? ─sí→ ejecuta Q y SAL
   no ↓
¿condición3? ─sí→ ejecuta R y SAL
   no ↓
ejecuta S (caso por defecto)
🎯 Regla de oro de la anidación

Pon primero la condición MÁS RESTRICTIVA. Si pones primero la condición más amplia, los casos restrictivos nunca se ejecutan porque "se los come" la primera condición.

1.4. El error clásico del orden incorrecto

Imagina este enunciado: "Si compras 50+ unidades, descuento 10%; si compras 20+, descuento 6%; sino, sin descuento".

❌ Orden incorrecto (de chico a grande)

SI n >= 20
    d = 0.06
SINO
    SI n >= 50
        d = 0.10
    SINO
        d = 0
    FIN_SI
FIN_SI

Problema: si compras 60 unidades, primero entra al "≥ 20" y aplica 6%. Nunca se evalúa el "≥ 50". El descuento del 10% es código muerto.

✅ Orden correcto (de grande a chico)

SI n > 50
    d = 0.10
SINO
    SI n >= 20
        d = 0.06
    SINO
        d = 0
    FIN_SI
FIN_SI

Funciona: compras 60 → "> 50" V → 10%. Compras 25 → "> 50" F → "≥ 20" V → 6%. Compras 10 → ambas F → 0%.

1.5. Reglas de indentación en SI anidados

Cada SI anidado se indenta 4 espacios más que el de afuera. Esto hace que el código se "vea como una pirámide" y sea legible:

📐 Patrón visual

Si un FIN_SI no está alineado verticalmente con su SI correspondiente, casi seguro tienes un error. Cuenta los SI y los FIN_SI: deben coincidir.

¿Qué es un ciclo? El gran panorama

💡 Intuición

Hasta ahora tu algoritmo solo ejecutaba cada instrucción una vez. Pero piensa: ¿cómo procesarías las notas de 50 alumnos sin repetir? ¿Tendrías que escribir 50 instrucciones Leer nota? Sería absurdo. Los ciclos permiten repetir el mismo bloque de instrucciones con datos diferentes, controlando cuándo parar.

2.1. Definición formal

Definición
Ciclo (bucle, lazo o loop)
Sección de código que se repite varias veces, controlada por una condición de parada. Cada repetición se llama iteración. El objetivo es colocar el bloque dentro del ciclo y agregar un elemento que permita salir para que el algoritmo continúe.

2.2. Las 3 estructuras de repetición

Pretest

🔄 MIENTRAS

Evalúa la condición al inicio. Puede ejecutarse 0 veces si la condición es F desde el principio.

  • • Para repetir mientras X
  • • Cantidad desconocida
  • • Validación de datos
Posttest

🔁 REPETIR-HASTA

Evalúa la condición al final. Garantiza que el bloque se ejecute al menos 1 vez.

  • • Para repetir hasta X
  • • Garantía de 1 ejecución
  • • Menús, contraseñas
Conteo fijo

🔢 PARA (DESDE)

Repite un número exacto de veces. La variable de control se actualiza automáticamente.

  • • Cantidad conocida
  • • Procesar N elementos
  • • Tablas, listas

2.3. ¿Cuándo usar cada una? (decisión rápida)

🧭 Árbol de decisión mental
  1. ¿Sé exactamente cuántas veces debo repetir? → PARA.
  2. ¿La condición se evalúa antes de entrar (puede no ejecutarse)? → MIENTRAS.
  3. ¿Necesito ejecutar al menos una vez y revisar la condición después? → REPETIR-HASTA.

2.4. Lo que TODO ciclo necesita (no negociable)

Independientemente de la estructura, todo ciclo bien hecho tiene 4 partes:

  1. Inicialización: dar valor inicial a las variables de control (contador = 0, suma = 0).
  2. Condición de parada: una expresión lógica que algún día será F (o V para REPETIR).
  3. Cuerpo del ciclo: las instrucciones que se repiten.
  4. Actualización: cambiar algo dentro del ciclo para que la condición eventualmente termine. Si olvidas esto, tienes un bucle infinito.
⚠️ Bucle infinito

Un bucle que nunca termina porque la condición nunca cambia o nunca se cumple la salida. Es el error más temido en programación. Solución: SIEMPRE asegúrate de que algo dentro del cuerpo modifique las variables de la condición.

3

MIENTRAS (while): "repite mientras se cumpla"

Definición
Estructura MIENTRAS
Estructura de repetición controlada por una condición al inicio. Si la condición es V, ejecuta el bloque y vuelve a evaluar. Si es F, sale. Si la condición es F desde el comienzo, el bloque NO se ejecuta nunca.

3.1. Sintaxis del pseudocódigo (curso)

MIENTRAS condición HACER
    // bloque P (se repite mientras condición sea V)
    instrucción_1
    instrucción_2
    actualización_de_variables   // ¡crítico para no entrar en bucle infinito!
FIN_MIENTRAS

3.2. Diagrama de flujo

graph TB I([Inicio del ciclo]) C{"¿condición?"} P["bloque P + actualizar variables"] F([Continúa el algoritmo]) I --> C C -->|V| P P --> C C -->|F| F style I fill:#34d399,stroke:#059669,color:#000 style F fill:#94a3b8,stroke:#475569,color:#000 style C fill:#fbbf24,stroke:#d97706,color:#000 style P fill:#a78bfa,stroke:#6d28d9,color:#fff
📊 La flecha de P regresa al rombo: por eso "se repite". La salida solo es por la rama F.

3.3. Las 4 partes del MIENTRAS (siempre)

// 1. INICIALIZACIÓN (antes del MIENTRAS)
contador = 0
Leer n          // "priming read": lectura inicial

// 2. CONDICIÓN DE PARADA (en el MIENTRAS)
MIENTRAS n > 0 HACER
    // 3. CUERPO (instrucciones que se repiten)
    contador = contador + 1
    // 4. ACTUALIZACIÓN (cambia algo de la condición)
    Leer n
FIN_MIENTRAS

3.4. El patrón "priming read" (lectura inicial)

🎓 Concepto avanzado pero esencial

Cuando un MIENTRAS depende de un dato leído del usuario, necesitas leerlo antes del ciclo. Es como "preparar la primera carta" antes de que comience el juego.

Patrón: leer fuera → entrar al MIENTRAS → procesar → leer de nuevo dentro → repetir.

Sin priming read tendrías que escribir el ciclo de forma rara. Con priming read, el código fluye naturalmente:

// Cuento positivos hasta que ingrese un negativo
cont = 0
Leer n                 // PRIMING READ (1ª lectura, fuera del ciclo)
MIENTRAS n > 0 HACER
    cont = cont + 1
    Leer n             // LECTURA SUBSIGUIENTE (dentro)
FIN_MIENTRAS

3.5. Característica clave: puede no ejecutarse nunca

Si la condición es F desde el principio, el cuerpo del MIENTRAS jamás se ejecuta:

n = 5
MIENTRAS n <= 4 HACER
    // nunca entra: 5 NO es ≤ 4
    ...
FIN_MIENTRAS
// el algoritmo continúa aquí sin haber ejecutado nada

Esto puede ser una ventaja o un defecto según el problema. Si quieres garantizar al menos una ejecución, usa REPETIR-HASTA (siguiente bloque).

4

REPETIR-HASTA: "ejecuta primero, pregunta después"

Definición
Estructura REPETIR-HASTA
Estructura de repetición controlada por una condición al FINAL. Ejecuta el bloque, luego evalúa la condición. Si es F, vuelve a repetir; si es V, sale. Garantiza que el cuerpo se ejecute al menos UNA vez.

4.1. Sintaxis del pseudocódigo (curso)

REPETIR
    // bloque P (se ejecuta y luego se pregunta)
    instrucción_1
    instrucción_2
HASTA QUE condición

4.2. Diagrama de flujo

graph TB I([Inicio del ciclo]) P["bloque P"] C{"¿condición?"} F([Continúa el algoritmo]) I --> P P --> C C -->|F| P C -->|V| F style I fill:#34d399,stroke:#059669,color:#000 style F fill:#94a3b8,stroke:#475569,color:#000 style C fill:#fbbf24,stroke:#d97706,color:#000 style P fill:#a78bfa,stroke:#6d28d9,color:#fff
📊 La diferencia con MIENTRAS: P está antes del rombo. Por eso siempre se ejecuta al menos una vez.

4.3. La gran trampa: condición invertida

⚠️ Cuidado al traducir entre MIENTRAS y REPETIR

MIENTRAS sigue mientras la condición es VERDADERA. REPETIR sigue mientras la condición es FALSA (sale cuando es V).

Por eso, al traducir un MIENTRAS a REPETIR-HASTA, debes invertir la condición:

🔄 MIENTRAS

MIENTRAS n <= 100 HACER
    ...
FIN_MIENTRAS

Sigue mientras n ≤ 100. Sale cuando n > 100.

🔁 REPETIR-HASTA equivalente

REPETIR
    ...
HASTA QUE n > 100

Sale cuando n > 100 (la negación de n ≤ 100).

4.4. MIENTRAS vs REPETIR: cuadro comparativo

CaracterísticaMIENTRASREPETIR-HASTA
Condición se evalúaAl inicio (pretest)Al final (posttest)
¿Cuándo continúa?Mientras condición es VMientras condición es F
¿Cuándo sale?Cuando es FCuando es V
Mínimo de ejecuciones0 (puede no entrar)1 (siempre entra)

4.5. Cuándo conviene REPETIR-HASTA

El caso clásico: validación de entrada. Imagina que pides un mes (1-12). Quieres preguntar al menos una vez, y seguir preguntando si la respuesta es inválida:

REPETIR
    Leer mes
    SI (mes < 1) OR (mes > 12)
        Mostrar "Valor entre 1 y 12"
    FIN_SI
HASTA QUE (mes >= 1) AND (mes <= 12)

Con MIENTRAS sería incómodo: tendrías que hacer una lectura previa fuera del ciclo. Con REPETIR-HASTA, todo queda dentro y el flujo es natural.

5

PARA (DESDE / for): "repite N veces exactas"

Definición
Estructura PARA (DESDE)
Estructura de repetición que ejecuta el bloque un número específico y conocido de veces. La variable de control (índice) se inicializa, se compara con el límite y se incrementa automáticamente en cada iteración.

5.1. Sintaxis del pseudocódigo (curso)

PARA i = 1 HASTA N HACER
    // bloque P se ejecuta N veces
    instrucciones que usan i (o no)
FIN_PARA

Variantes equivalentes que verás en otros materiales: DESDE i = 1 HASTA N o desde i ← 1 hasta N hacer (Joyanes). En el curso usamos PARA / FIN_PARA.

5.2. Diagrama de flujo

graph TB I([Inicio del ciclo]) A["i = 1 (inicialización)"] C{"¿i <= N?"} P["bloque P"] Inc["i = i + 1 (automático)"] F([Continúa el algoritmo]) I --> A --> C C -->|V| P --> Inc --> C C -->|F| F style I fill:#34d399,stroke:#059669,color:#000 style F fill:#94a3b8,stroke:#475569,color:#000 style C fill:#fbbf24,stroke:#d97706,color:#000 style P fill:#a78bfa,stroke:#6d28d9,color:#fff style A fill:#fbbf24,stroke:#d97706,color:#000 style Inc fill:#fbbf24,stroke:#d97706,color:#000
📊 Lo que hace ahorrar el PARA: la inicialización, comparación e incremento se manejan automáticamente.

5.3. ¿Qué pasa "por debajo" del PARA?

El PARA es azúcar sintáctica: hace en una línea lo que el MIENTRAS necesita en tres pasos. Compara:

🔢 PARA (compacto)

PARA i = 1 HASTA 10 HACER
    Mostrar i
FIN_PARA

🔄 MIENTRAS equivalente

i = 1                    // inicialización
MIENTRAS i <= 10 HACER     // condición
    Mostrar i
    i = i + 1          // actualización
FIN_MIENTRAS

Hacen lo mismo. Pero el PARA tiene 3 ventajas:

  1. Es más compacto (1 vs 5 líneas).
  2. Es imposible olvidar la actualización; el incremento es automático.
  3. Comunica la intención: "voy a repetir N veces" se ve a primera vista.

5.4. La variable de control "i"

Por convención (y para no confundirte con tus variables), la variable de control de un PARA suele llamarse i, j o k (de index en inglés). Tiene tres usos típicos:

📌 Uso 1: contador puro

No usas i dentro del bloque, solo te interesa repetir N veces.

PARA i = 1 HASTA 5 HACER
    Leer edad
    suma = suma + edad
FIN_PARA

📌 Uso 2: i como dato

Usas i para calcular o mostrar.

PARA i = 1 HASTA 10 HACER
    Mostrar N, " x ", i, " = ", N*i
FIN_PARA

5.5. Cuándo usar PARA vs MIENTRAS

🧭 Regla práctica
  • Cantidad CONOCIDA (al inicio sabes cuántas veces): PARA. Ej: "leer las notas de los 30 alumnos".
  • Cantidad DESCONOCIDA (depende del usuario o de los datos): MIENTRAS. Ej: "leer notas hasta que ingrese -1".
  • Ejecutar al menos 1 vez sí o sí: REPETIR-HASTA. Ej: "menú que se muestra hasta que el usuario elija salir".
6

Patrones clave: contadores, acumuladores y De Morgan

6.1. CONTADOR vs ACUMULADOR (concepto fundamental)

🔢 Contador

Variable que se incrementa o disminuye en una cantidad CONSTANTE cada iteración. Sirve para contar cuántas veces ocurre algo.

contador = contador + 1

Ejemplo: contar cuántos números positivos se ingresaron.

➕ Acumulador (totalizador)

Variable que suma sobre sí misma un conjunto de valores variables. Sirve para obtener una suma total.

suma = suma + N

Ejemplo: sumar todas las notas para calcular el promedio.

💡 La diferencia clave

El contador siempre suma el mismo valor (típicamente 1). El acumulador suma valores que cambian (las notas, los precios, etc.). Frecuentemente se usan juntos: un contador para saber cuántos elementos hay y un acumulador para tener su suma; con ambos calculas el promedio.

6.2. Inicialización: ¿0 o 1?

VariableInicializa en¿Por qué?
Contador que arranca de cero (cuántas veces ocurrió algo)= 0Si nunca ocurre, queda en 0 (correcto).
Contador de iteraciones en PARA "1 hasta N"= 1La variable de control empieza en 1.
Acumulador de suma= 0Identidad de la suma: x + 0 = x.
Acumulador de producto= 1Identidad del producto: x × 1 = x. ¡Si lo pones en 0, todo el producto da 0!

6.3. Leyes de De Morgan: invertir condiciones complejas

🧮 Las dos leyes
NOT (a AND b) ≡ (NOT a) OR (NOT b)
"No es cierto que A y B" = "No A, o no B"
NOT (a OR b) ≡ (NOT a) AND (NOT b)
"No es cierto que A o B" = "No A, y no B"

¿Para qué sirven? Son la herramienta más útil para convertir un MIENTRAS en un REPETIR-HASTA (que requiere invertir la condición).

📐 Demostración intuitiva

Imagina la condición "edad ≥ 18 AND tiene DNI" (entrada permitida). Su negación, "NO se permite la entrada", se cumple si:

  • NO tiene 18+ años (es menor de edad), O
  • NO tiene DNI

Es decir: (edad < 18) OR (NOT tieneDNI). Esto es exactamente lo que dice De Morgan.

📐 Verificación con tabla de verdad

aba AND bNOT (a AND b)(NOT a) OR (NOT b)
VVVFF
VFFVV
FVFVV
FFFVV

Las dos columnas finales son idénticas en todas las filas → las expresiones son equivalentes. ✓

6.4. De Morgan en la práctica: pasar MIENTRAS a REPETIR-HASTA

Recuerda: REPETIR-HASTA sale cuando la condición es V; MIENTRAS continúa cuando es V. Por eso, al traducir, hay que negar la condición:

🔄 Versión MIENTRAS

MIENTRAS (n >= 1) AND (n <= 12) HACER
    ...
FIN_MIENTRAS

🔁 Versión REPETIR-HASTA equivalente

REPETIR
    ...
HASTA QUE (n < 1) OR (n > 12)

¿Por qué? Aplico De Morgan: NOT[(n≥1) AND (n≤12)] = (n<1) OR (n>12). El "AND" se convierte en "OR" y cada comparación se invierte.

Ejercicios resueltos paso a paso

Cada ejercicio sigue tu metodología completa. Los 5 primeros son ejemplos resueltos del archivo S5; los últimos 3 refuerzan los patrones clave (anidación, descomposición y validación).

EJ. 1Contar números enteros positivos hasta ingresar uno negativo (MIENTRAS)📚 S5 ejemplo
📝 Enunciado

Elabore un algoritmo que cuente los números enteros positivos que se ingresen. El programa finaliza cuando se ingrese un número negativo.

🎯 ¿Qué nos están pidiendo?

Procesar una cantidad desconocida de números (no sabemos cuántos ingresará el usuario), incrementando un contador cada vez que el número sea positivo, y parar cuando ingrese un negativo.

🧠 Razonamiento: ¿qué estructura uso?

MIENTRAS, porque:

  • No conozco la cantidad de iteraciones (el usuario decide cuándo parar) → no es PARA.
  • Si el usuario ingresa un negativo de entrada, el ciclo NO debe ejecutarse (contador = 0) → MIENTRAS, no REPETIR-HASTA.

🧠 Patrón clave: priming read

Necesito leer el primer número antes del MIENTRAS para que la condición pueda evaluarse. Dentro del ciclo leo el siguiente número.

🧠 Identificar variables (leyenda)

🏷️ Leyenda
  • n = número leído (entrada, Entero)
  • cont = contador de números positivos (salida, Entero)

🪜 Pseudocódigo

INICIO
    Entero: n, cont
    cont = 0           // inicialización del contador
    Leer n             // priming read (1ª lectura, fuera del ciclo)
    MIENTRAS n > 0 HACER
        cont = cont + 1   // solo cuento si el número fue positivo
        Leer n         // lectura subsiguiente (dentro del ciclo)
    FIN_MIENTRAS
    Mostrar cont
FIN

📊 Observa los DOS bloques de "leer n": uno antes del rombo (priming) y otro dentro del ciclo.

🧪 Prueba de escritorio

Ingresos: 5, 3, 8, -1 (en ese orden)

IterInstrucciónncont
—cont = 0—0
—Leer n (priming) → ingresa 550
1SI 5 > 0? V → cont = 0+1, leer n → ingresa 331
2SI 3 > 0? V → cont = 1+1, leer n → ingresa 882

Caso especial: si el primer número es −5 directamente, el MIENTRAS evalúa V−5>0 → F y nunca entra al cuerpo. cont sigue en 0 y se imprime 0. ✓

✅ Resultado

Para los datos 5, 3, 8, -1 → contador = 3.

💡 Lección clave

El priming read es esencial cuando la condición del MIENTRAS depende de un dato del usuario. Sin él, el algoritmo no sabría qué evaluar la primera vez. Patrón: leer fuera → MIENTRAS → procesar → leer dentro → repetir.

EJ. 2Suma de números pares positivos hasta 100 (MIENTRAS y REPETIR)📚 S5 ejemplo
📝 Enunciado

Obtenga la suma de los números pares positivos hasta 100. Resuelva primero con MIENTRAS y luego con REPETIR-HASTA. Elabore los diagramas de flujo para ambos casos.

🎯 ¿Qué nos están pidiendo?

Calcular 2 + 4 + 6 + ... + 100 (la suma de pares de 2 a 100). Aquí sí conocemos cuántas iteraciones (50), pero el ejercicio nos pide usar MIENTRAS y REPETIR para practicar.

🧠 Razonamiento clave

  • Necesito un acumulador (Suma) para ir sumando.
  • Necesito una variable de control (n) que comienza en 2 y avanza de 2 en 2 (para saltar los impares).
  • El ciclo termina cuando n pase de 100.

🧠 Identificar variables (leyenda)

🏷️ Leyenda
  • n = número par actual (control, Entero)
  • Suma = acumulador de pares (salida, Entero)

🪜 Versión 1 · Pseudocódigo MIENTRAS

INICIO
    Entero: n, Suma
    Suma = 0          // inicializo el acumulador en 0 (identidad de la suma)
    n = 2             // arranco en el primer par positivo
    MIENTRAS n <= 100 HACER
        Suma = Suma + n   // acumulo el par actual
        n = n + 2     // avanzo al siguiente par
    FIN_MIENTRAS
    Mostrar Suma
FIN

🪜 Versión 2 · Pseudocódigo REPETIR-HASTA

INICIO
    Entero: n, Suma
    Suma = 0
    n = 2
    REPETIR
        Suma = Suma + n
        n = n + 2
    HASTA QUE n > 100     // negación de n <= 100 (sale cuando n > 100)
    Mostrar Suma
FIN
🎯 Punto clave: la inversión

Observa cómo n <= 100 (MIENTRAS, mientras es V) se convierte en n > 100 (REPETIR, sale cuando es V). Es la negación directa.

🔷 Diagrama de flujo MIENTRAS

EJ. 8¿Es primo? (MIENTRAS con bandera/flag)🧠 ejercicio integrador
📝 Enunciado

Lea un número entero positivo n > 1 y determine si es primo (solo divisible por 1 y por sí mismo) o compuesto.

🎯 ¿Qué nos están pidiendo?

Detectar si n tiene algún divisor entre 2 y n−1. Si encuentro alguno → es compuesto. Si recorro todos sin encontrar ninguno → es primo.

🧠 Razonamiento clave: el patrón "bandera" (flag)

Uso una variable esPrimo que arranca en V (asumo que es primo). Si en algún momento encuentro un divisor, la cambio a F. Al final, si sigue en V, no encontré ningún divisor → es primo.

🧠 ¿Por qué MIENTRAS y no PARA?

Porque puedo parar antes si ya encontré un divisor (no necesito seguir buscando). Con PARA tendría que recorrer todos los números obligatoriamente. Con MIENTRAS puedo poner una condición compuesta que me deje salir temprano.

🧠 Identificar variables (leyenda)

🏷️ Leyenda
  • n = número a evaluar (entrada, Entero)
  • i = posible divisor que voy probando (control, Entero)
  • esPrimo = bandera lógica (intermedio, Lógico)

🪜 Pseudocódigo

INICIO
    Entero: n, i
    Lógico: esPrimo
    Leer n
    esPrimo = V      // asumo que es primo
    i = 2            // arranco probando divisores desde 2
    MIENTRAS (i < n) AND (esPrimo = V) HACER
        SI n mod i = 0     // si i divide a n exactamente
            esPrimo = F      // encontré un divisor → no es primo
        FIN_SI
        i = i + 1
    FIN_MIENTRAS
    SI esPrimo = V
        Mostrar n, " es primo"
    SINO
        Mostrar n, " es compuesto"
    FIN_SI
FIN
🧮 ¿Qué es n mod i?

mod es el resto de la división entera. Si n mod i = 0 significa que i divide exactamente a n (sin resto). Por ejemplo: 10 mod 5 = 0 (5 divide a 10), 10 mod 3 = 1 (3 no divide exactamente).

Glosario rápido

Bucle (ciclo, loop)
Sección de código que se repite varias veces controlada por una condición.
Iteración
Cada pasada o ejecución del cuerpo del bucle.
MIENTRAS (while)
Bucle pretest: condición al inicio. Puede ejecutarse 0 veces.
REPETIR-HASTA
Bucle posttest: condición al final. Ejecuta al menos 1 vez.
PARA (for, desde)
Bucle de conteo fijo con variable de control automática.
Variable de control
Variable que controla la cantidad de iteraciones (típicamente i, j, k).
Contador
Variable que se incrementa en cantidad constante (típicamente +1) cada iteración.
Acumulador
Variable que suma sobre sí misma valores variables para totalizar.
Inicialización
Dar valor inicial a contador/acumulador antes del ciclo (típicamente 0 o 1).
Actualización
Cambiar las variables dentro del ciclo para que la condición eventualmente termine.
Bucle infinito
Bucle que nunca termina por error en su diseño (olvido de actualización).
Priming read
Lectura inicial antes del MIENTRAS para que la condición pueda evaluarse.
Pretest
Bucle que evalúa la condición al inicio (MIENTRAS, PARA).
Posttest
Bucle que evalúa la condición al final (REPETIR-HASTA).
Bandera (flag)
Variable lógica (V/F) que registra si ocurrió un evento dentro del ciclo.
SI anidado
Un SI dentro de la rama V o F de otro SI.
Filtro en cascada
Patrón de SI anidados donde cada SINO lleva a la siguiente pregunta.
Ley de De Morgan
NOT(a AND b) = (NOT a) OR (NOT b); NOT(a OR b) = (NOT a) AND (NOT b).
mod (módulo)
Operador que devuelve el resto de la división entera.
Identidad de la suma/producto
0 para sumar (no afecta), 1 para multiplicar (no afecta). Por eso se inicializan así.

Autoevaluación · 12 preguntas

Responde mentalmente, luego abre la respuesta. Si fallas más de 3, repasa los bloques correspondientes.

¿Cuáles son las 3 estructuras de repetición?
Ver respuesta
MIENTRAS (pretest), REPETIR-HASTA (posttest) y PARA (conteo fijo).
¿Cuál es la diferencia entre MIENTRAS y REPETIR-HASTA?
Ver respuesta
MIENTRAS: evalúa al inicio, puede ejecutarse 0 veces. REPETIR-HASTA: evalúa al final, ejecuta siempre al menos 1 vez. Además, MIENTRAS continúa cuando V; REPETIR-HASTA sale cuando V (condiciones invertidas).
¿Qué es un contador y qué es un acumulador?
Ver respuesta
Contador: suma una cantidad constante (típicamente 1) cada iteración. Acumulador: suma valores variables sobre sí mismo. Ejemplo: cont = cont + 1 vs suma = suma + N.
¿En qué valor inicializo un acumulador de productos?
Ver respuesta
En 1, no en 0. Si lo inicializas en 0, el primer producto da 0 y todo se vuelve 0. El 1 es la identidad del producto: x × 1 = x.
Aplica De Morgan: ¿cuál es la negación de (a >= 1) AND (a <= 12)?
Ver respuesta
(a < 1) OR (a > 12). El AND se vuelve OR, y cada comparación se invierte.
¿Qué es un priming read y para qué sirve?
Ver respuesta
Una lectura inicial antes del MIENTRAS. Sirve cuando la condición depende de un dato del usuario: lo lees primero, evalúas, y luego dentro del ciclo lees el siguiente.
¿Cuándo conviene usar PARA?
Ver respuesta
Cuando se conoce exactamente cuántas iteraciones se harán (cantidad fija o leída al inicio). Ejemplo: "procesar las notas de los 50 alumnos".
En un SI anidado para 3 rangos (descuentos), ¿en qué orden van las condiciones?
Ver respuesta
De más restrictiva a menos restrictiva. Si pones primero la condición amplia, los casos restrictivos nunca se ejecutan.
¿Qué es un bucle infinito y cómo lo evito?
Ver respuesta
Un bucle que nunca termina por error en el diseño. Se evita asegurando que siempre haya una instrucción dentro del cuerpo que modifique las variables de la condición (ej: n = n + 1).
¿Qué es una bandera (flag) en un algoritmo?
Ver respuesta
Una variable lógica (V/F) que registra si ocurrió un evento durante el ciclo. Se usa típicamente cuando uno quiere saber "¿existió/se cumplió X durante todo el recorrido?".
¿Qué hace 15 mod 4?
Ver respuesta
3. Es el resto de dividir 15 entre 4 (15 = 4×3 + 3).
Las 4 partes obligatorias de cualquier ciclo:
Ver respuesta
1) Inicialización de variables. 2) Condición de parada. 3) Cuerpo del ciclo. 4) Actualización de las variables de la condición.
